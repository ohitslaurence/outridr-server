## Findings considered and rejected / deferred

- **Split `lib/server.mjs` (663 lines, five concerns) into modules** — deferred:
  real but low-leverage now; the plan-001 integration tests are deliberately
  written against HTTP/WS so a future split is safe and mechanical.
- **Synchronous fs I/O in the `/session` request path** (`statSync`/`readSync`
  of ≤512 KiB windows) — not worth doing: single-user local-disk workload;
  blocking time is microseconds-to-low-ms. Revisit only if multi-client use
  appears.
- **No negative caching in `findSessionFile`** (unknown session id rescans all
  project dirs per request) — not worth doing at current scale; note kept here
  for the record.
- **`?token=` auth in query strings** — by design (WS clients that can't set
  headers); documented in README. Not a finding.
- **`/exec` passes client-supplied args to the configured binary** — by
  design and documented ("runs exactly the one configured binary with
  client-supplied args", opt-in, tailnet + token gated). The security posture
  depends on the operator's choice of binary; README already says so.
- **herdr socket has no auth of its own** — documented security model
  (tailnet ACLs + optional token); out of this repo's control.
- **`launchctl load/unload` deprecated in favor of `bootstrap/bootout`** —
  deferred: still functional on current macOS; revisit when Apple breaks it.
- **Plist/unit path escaping** (XML-significant chars in install paths) —
  deferred: pathological install paths only; would surface immediately at
  install time.
- **Expo receipts API polling and >100-message batching** — deferred; see
  plan 005 maintenance notes.
- **Tailscale IP change while running leaves a stale bind** — deferred; see
  plan 004 maintenance notes.

From the 2026-07-20 audit:

- **`readBody` (`lib/http-util.mjs:34-46`) destroys over-limit requests
  without a 413 response** — accepted: bodies on those routes are small JSON;
  same worst case as a dropped connection.
- **`npm install -g npm@latest` unpinned in release.yml** — accepted: npm
  trusted publishing requires npm ≥ 11.5.1, newer than node 22 bundles;
  revisit when the setup-node default catches up.
- **All `/transcribe` findings (no tests, no docs, no fetch timeout, broken
  413 flush, `OUTRIDR_GROQ_API_KEY=""` config edge)** — mooted 2026-07-20:
  the operator confirmed the outridr app now calls Groq directly, so plan 012
  removes the endpoint entirely instead of hardening it. If server-side
  transcription ever returns, prefer a local-command backend
  (`transcribe.command` → e.g. whisper.cpp) over a cloud-key proxy.
